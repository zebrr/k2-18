# K2-18: Семантические графы в образовании

## Почему линейный учебник не отражает структуру знаний

Учебник устроен как последовательность страниц. Но знания нелинейны: чтобы понять quicksort, нужно знать sorting, а чтобы понять sorting — понять массивы. Это цепочка предпосылок, которую учебник прячет внутри себя.

Раньше извлечь эту структуру из текста была ручная работа методиста — дорогая и трудоёмкая. Теперь это область **knowledge extraction** — автоматическое извлечение структуры знаний из текста с помощью LLM.

---

## Семантический граф

Граф — узлы и рёбра. Узлы у нас трёх типов: учебные фрагменты (Chunks), концепты (Concepts), задания (Assessments).

Но рёбра — это ключевое. Если ребро просто "связано" — это топологический граф, структура без смысла. Когда ребро типизировано — это **семантический** граф, и тут появляется прагматика: не просто "A связано с B", а **почему** A связано с B и что это означает для обучения.

`PREREQUISITE` — нельзя понять B без A. `ELABORATES` — B раскрывает A глубже. `EXAMPLE_OF` — A иллюстрирует B. `TESTS` — задание проверяет понимание концепта.

Именно типы рёбер несут педагогический смысл графа.

## Pipeline

Текст → граф за пять шагов.

**Slicer** делит исходный текст на семантически связные части, учитывая границы абзацев, заголовки, ссылки, формулы, код и т.п. Это решает проблему контекстного окна LLM.

**iText2KG Concepts** — последовательно обрабатывает все части и строит словарь понятий с каноническими определениями и алиасами.

**iText2KG Graph** — строит граф, используя готовый словарь как ключевой связующий элемент. Знаем понятия заранее = задача сужается до ~расстановки рёбер.

**Dedup** — убирает семантические дубликаты через embeddings и FAISS: одна и та же идея, повторенная в разных частях текста, схлопывается в один узел.

**Refiner** — двухпроходный анализ (forward/backward) для выявления дальних зависимостей между понятиями из разных частей учебника. Без этого шага `PREREQUISITE` между первой и десятой главой просто не появится.

Шаги 2, 3, 5 требуют доменной адаптации.

## Метрики и кластеры

Граф построен — теперь нужно понять его структуру. Мы считаем 12 метрик на каждом узле.

Часть из них стандартные графовые: **PageRank** (важность узла с учётом того, кто на него ссылается), **betweenness centrality** (узел как мост между частями графа), **closeness** (насколько легко из узла добраться до остальных).

Часть — специфичные для обучения: **prerequisite_depth** — уровень в иерархии предпосылок (0 = основы, выше = продвинутые темы). **learning_effort** — накопленная сложность с учётом всей цепочки предпосылок. **educational_importance** — PageRank, но только по образовательным рёбрам (PREREQUISITE, ELABORATES, TESTS, EXAMPLE_OF).

Кластеризация — алгоритм Louvain на неориентированной проекции графа. Даёт тематические блоки и **bridge_score** — насколько узел соединяет разные кластеры.

## Применения

Несколько примеров сценариев — на практике их значительно больше.

**Учебный путь**: минимальный маршрут от «переменные» до «рекурсия» с учётом цепочек PREREQUISITE. Можно адаптировать — для слабого студента путь через примеры и низкую сложность, для сильного — прямой.

**Обнаружение пробелов**: студент прошёл узлы A и B, хочет перейти к C — система находит все недостающие предпосылки по графу.

**Анализ структуры курса**: какие концепты центральные (высокий PageRank), где узкие места (высокий betweenness), как тематически сгруппирован материал (кластеры). То, что методист раньше делал интуитивно — теперь измеримо.
